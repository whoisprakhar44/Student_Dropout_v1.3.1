======================================================================
SLIDE 2: PROJECT OBJECTIVE
======================================================================

Objective

To develop an integrated system that supports early identification of
students at risk of dropping out by combining:

- Student-level academic and attendance data
- School-level operational data
- Risk scoring and dashboard analytics
- Detailed student profiles and intervention guidance
- Natural-language querying of the curated school database

Expected Outcome

Enable education stakeholders to identify risk patterns, review student
details, and make data-informed intervention decisions.


======================================================================
SLIDE 3: SOLUTION OVERVIEW
======================================================================

The current solution is a local full-stack application with the following
components:

- Web dashboard for monitoring student risk indicators
- FastAPI backend for application services and REST APIs
- SQLite database populated with representative sample data
- Curated YAML-based school data model
- LangGraph agent for natural-language query orchestration
- MCP tools for schema retrieval and read-only SQL execution
- Milvus Lite vector store for schema retrieval support

Primary Use Cases

- Monitor overall student risk distribution
- Browse and review individual student records
- Analyze attendance and academic performance
- Query school data using natural-language questions

SLIDE 9: AI-ASSISTED QUERY ARCHITECTURE
======================================================================

Natural-Language Query Flow

User Question
     |
     v
Web Interface / POST /ask
     |
     v
LangGraph Agent
     |
     +--> MCP Schema Retrieval Tool
     |    Retrieves relevant curated table definitions and joins
     |
     +--> MCP SQL Execution Tool
          Executes read-only SQLite SELECT queries
     |
     v
Structured Answer Returned to the User

Key Improvement

The language model can decide when schema retrieval is required.
Retrieval is no longer forced before every query.

SLIDE 12: CURRENT LIMITATIONS
======================================================================

The present version is suitable for demonstration and further development.
The following items remain before production deployment:

- Authentication is configured for demonstration use only
- Sessions are stored in memory
- SQLite is intended for local testing and prototyping
- Ollama and embedding models must be available for AI-assisted querying
- Production security controls are not yet implemented
- HTTPS, rate limiting, audit logging, and deployment monitoring are pending
- Hive execution is included only as a disabled future-integration stub
