Ask API Contract
----------------
`POST /ask` accepts a natural-language question and returns only:

```json
{
  "sql": "SELECT ...",
  "result": []
}
```

`sql` is the SQL generated and executed by the agent.
`result` is always a list of row objects returned by that SQL.

Supported ask behavior:
- Count questions return one result row.
- Row/list questions return row objects in `result`.
- Prompts like `Show 5 students` should generate SQL with `LIMIT 5`.
- Prompts like `Show 20 schools` should generate SQL with `LIMIT 20`.
- Prompts like `Show the top school` should return one ranked school row, normally ranked by student count unless another metric is requested.

Target-System Test Steps
------------------------
Run these steps on the device/server where Ollama is available.

1. Activate the project virtual environment.

2. Install dependencies inside the venv only:
   `python -m pip install -r requirements.txt`

3. Ensure Ollama is running and these models are available:
   - chat model from `.env`, currently `qwen3.5:2b`
   - embedding model from `MCP/mcp_rag.yaml`, currently `nomic-embed-text`

4. Regenerate curated sample SQLite DB:
   `python create_schema.py`

5. Rebuild Milvus Lite schema index:
   `python MCP/build_milvus_index.py`

6. Start the API server:
   `python -m uvicorn app:app --host 0.0.0.0 --port 8000`

Bruno Test Requests
-------------------
Use `Content-Type: application/json` for all `/ask` requests.

Health:
`GET http://<server-host>:8000/health`

Count query:
`POST http://<server-host>:8000/ask`

Body:
```json
{
  "question": "How many students are in the database?"
}
```

Expected response shape:
```json
{
  "sql": "SELECT COUNT(*) AS total_students FROM citizen_student",
  "result": [
    {
      "total_students": 1000
    }
  ]
}
```

Student rows:
```json
{
  "question": "Show 5 students"
}
```

Expected:
- SQL includes `LIMIT 5`.
- `result` contains up to 5 student row objects.

School rows:
```json
{
  "question": "Show 20 schools"
}
```

Expected:
- SQL includes `LIMIT 20`.
- `result` contains up to 20 school row objects.

Top school:
```json
{
  "question": "Show the top school"
}
```

Expected:
- SQL groups/ranks schools and includes `LIMIT 1`.
- `result` contains one school row.
